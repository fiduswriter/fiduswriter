# Upgrading Plugins from Fidus Writer 4.1.x to 5.0.0

This guide covers the breaking changes plugin developers need to address when
migrating a plugin from Fidus Writer 4.1.x to 5.0.0.

---

## Breaking-change summary

| Area | What changed |
|---|---|
| **Document schema / importers / exporters** | Moved to the standalone npm package `@fiduswriter/document` |
| **fwtoolkit (Fidus Writer Toolkit)** | All modules previously under `base/static/js/modules/common` (dialogs, menus, network, settings, etc.) are now in the separate package `fwtoolkit` |
| **Local proxy files** | The one-line `export * from "@fiduswriter/document/..."` re-exports in `document/static/js/modules/{schema,exporter,importer,citations}` and `bibliography/static/js/modules/bibliography/schema` have been removed |
| **Plugin imports** | Relative imports such as `../../../exporter/docx` or `../schema/document` must be replaced with package imports |
| **Schema JSON export** | The Django `export_schema` management command is removed; `schema.json` is now generated by the npm package |
| **JS tests** | The Jest tests for importers/exporters moved into `@fiduswriter/document`; the Django wrapper `document.tests.test_import_export` is removed |
| **Test fixture generation** | `generate_import_testfiles` now writes fixtures into the `@fiduswriter/document` package tree |
| **Dependencies** | `@fiduswriter/document` and `fwtoolkit` are installed from npm |

---

## 1. Document schema, importers and exporters are now an npm package

### What changed

All document-related conversion and schema code now lives in the
`@fiduswriter/document` package instead of inside the main repository under
`document/static/js/modules/{schema,exporter,importer,citations}` and
`bibliography/static/js/modules/bibliography/schema`.

The packages are published to npm (e.g. `@fiduswriter/document@^0.1.0-alpha.2`
and `fwtoolkit@^0.1.0-alpha.6`) and declared as dependencies in
`fiduswriter/base/package.json5`:

```json5
{
    dependencies: {
        "@fiduswriter/document": "^0.1.0-alpha.2",
        "fwtoolkit": "^0.1.0-alpha.6",
        // ...
    }
}
```

There is no local file dependency any more; `pnpm install` resolves the packages
from the npm registry.

### What you need to do

Make sure your development environment installs the new dependencies:

```bash
cd fiduswriter
python manage.py npm_install
# or simply
python manage.py transpile --force
```

If your plugin uses the shared helpers but not any document-specific code, you
can depend on `fwtoolkit` alone:

```json5
{
    dependencies: {
        "fwtoolkit": "^0.1.0-alpha.6"
    }
}
```

`fwtoolkit` is published under the **LGPL-3.0-or-later** license so it can be reused
in applications with different licenses. `@fiduswriter/document` remains
**AGPL-3.0**.

## 2. Remove relative imports to moved modules

### What changed

The old proxy files that re-exported package members have been deleted. Any
plugin code that imported a moved module through a relative path will now fail
to resolve.

### Migration examples

#### Schema imports

**Old (4.1.x)**

```javascript
import {docSchema} from "../schema/document"
import {randomHeadingId} from "../schema/common"
import {LANGUAGES} from "../schema/const"
```

**New (5.0.0)**

```javascript
import {docSchema} from "@fiduswriter/document/schema/document"
import {randomHeadingId} from "@fiduswriter/document/schema/common"
import {LANGUAGES} from "@fiduswriter/document/schema/const"
```

#### Exporter / importer imports

**Old (4.1.x)**

```javascript
import {DOCXExporter} from "../../../exporter/docx"
import {ZipFileCreator} from "../../../exporter/tools/zip"
import {DocxConvert} from "../../../importer/docx/convert"
```

**New (5.0.0)**

```javascript
import {DOCXExporter} from "@fiduswriter/document/exporter/docx"
import {ZipFileCreator} from "@fiduswriter/document/exporter/tools/zip"
import {DocxConvert} from "@fiduswriter/document/importer/docx/convert"
```

#### Bibliography schema imports

**Old (4.1.x)**

```javascript
import {cslBibSchema} from "../../bibliography/schema/csl_bib"
```

**New (5.0.0)**

```javascript
import {cslBibSchema} from "@fiduswriter/document/bibliography/csl_bib"
```

#### fwtoolkit imports (all `common` modules)

All modules that used to live under `base/static/js/modules/common` — dialogs,
menus, network helpers, settings, file dialogs, etc. — have been moved into the
`fwtoolkit` package. Any relative import that reached them through a `common`
path must be updated.

**Old (4.1.x)**

```javascript
import {Dialog, postJson} from "../common"
import {moveFile} from "../../common/file/tools"
import {escapeText} from "../../../common"
```

**New (5.0.0)**

```javascript
import {Dialog, postJson} from "fwtoolkit"
import {moveFile} from "fwtoolkit/file/tools"
import {escapeText} from "fwtoolkit"
```

The document-specific helpers that used to be exported as
`@fiduswriter/document/common` are also in `fwtoolkit` now:

```javascript
import {escapeText, addAlert, postJson} from "fwtoolkit"
```

When a plugin only needs shared helpers, depend on `fwtoolkit` directly. Use
`@fiduswriter/document` only when you also need document-specific schema or
conversion code.

## 3. Common plugin patterns

### Re-exporting a package module from your plugin

If your plugin previously wrapped a core exporter, remove the wrapper and
import the package directly:

**Old (4.1.x)**

```javascript
// myplugin/static/js/modules/myplugin/exporter/docx.js
export * from "../../exporter/docx"
```

**New (5.0.0)**

```javascript
// Import @fiduswriter/document/exporter/docx directly where you need it.
// A separate wrapper file is no longer required.
```

### Extending an exporter class

Book and plugin exporters that subclass core exporters should import the base
class from the package:

**Old (4.1.x)**

```javascript
import {DOCXExporterRender} from "../../../exporter/docx/render"
```

**New (5.0.0)**

```javascript
import {DOCXExporterRender} from "@fiduswriter/document/exporter/docx/render"
```

## 4. Schema JSON generation

### What changed

The Django management command `document/management/commands/export_schema.py`
has been removed. The JSON serialization of the schema is now generated by the
npm package:

- `npm run build-schema` inside `@fiduswriter/document` produces `schema.json`.
- `prepare` / `prepublishOnly` run the schema export automatically before
  publish/install.
- `copy_schema` in `base/package.json5` copies the generated
  `node_modules/@fiduswriter/document/schema.json` to
  `static-libs/json/schema.json` for the Python backend.

### What you need to do

Nothing, unless your plugin relied on running `export_schema` manually. In that
case, run the package script instead:

```bash
cd ../fiduswriter-document
npm run build-schema
```

The Python backend reads the schema from `static-libs/json/schema.json` as
before.

## 5. Tests

### What changed

The Jest tests for importers and exporters now live inside
`fiduswriter-document/test/` and are run with:

```bash
cd fiduswriter-document
npm test
```

The Django test wrapper `document.tests.test_import_export` has been removed,
and the corresponding entry has been dropped from the CI matrix.

### What you need to do

- Move any plugin-specific import/export Jest tests into your plugin’s own test
  setup.
- Do not depend on the core `document.tests.test_import_export` test case; it
  no longer exists.

---

## 6. Regenerating import test fixtures

If you use the management command that generates `comprehensive-test.docx` and
`comprehensive-test.odt` for import testing, its default output directory has
moved into the package:

```
fiduswriter-document/test/importer/fixtures/
```

You can still regenerate them explicitly:

```bash
cd fiduswriter
python manage.py generate_import_testfiles --output-dir \
    ../fiduswriter-document/test/importer/fixtures
```

---

## Checklist

- [ ] Replace all relative imports that pointed to
      `document/static/js/modules/{schema,exporter,importer,citations}` or
      `bibliography/static/js/modules/bibliography/schema` with imports from
      `@fiduswriter/document/...`.
- [ ] Replace imports from `@fiduswriter/document/common` (and any relative
      `../common` paths that used the shared helpers) with imports from
      `fwtoolkit`.
- [ ] Delete any plugin wrapper files that only re-export a moved core module.
- [ ] Run `python manage.py transpile --force` and fix any module-resolution
      errors.
- [ ] Update CI/plugin test commands to run `npm test` inside
      `fiduswriter-document` instead of the removed Django test case.
- [ ] If you regenerate import test fixtures, use the new package fixture path.
